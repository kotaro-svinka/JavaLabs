Реализация и использование Хэш-таблиц на Java

Данный проект состоит из двух частей, демонстрирующих принципы работы хэш-таблиц: создание собственной структуры данных "с нуля" и использование встроенных инструментов Java Collection Framework.

## Содержание
Часть 1: Собственная реализация HashTable
Часть 2: Использование HashMap для учета склада
Основные концепции

Часть 1: Собственная реализация HashTable

В этой части реализована классическая хэш-таблица с использованием метода цепочек для разрешения коллизий.

Ключевые особенности:
Универсальность: Использование Generics (дженериков) `<K, V>` позволяет работать с любыми типами данных.
Разрешение коллизий: Каждая ячейка массива представляет собой `LinkedList`, в котором хранятся объекты внутреннего класса `Entry`.
Безопасность типов: Применение аннотации `@SuppressWarnings("unchecked")` для корректной работы с массивами обобщенных типов.

Реализованные методы:
- `put(K key, V value)` — вставка или обновление значения.
- `get(K key)` — поиск значения по ключу за среднее время $O(1)$.
- `remove(K key)` — безопасное удаление элемента (защита от `ConcurrentModificationException`).
- `size()` — получение текущего количества элементов.


Часть 2: Использование HashMap для учета склада

Вторая часть демонстрирует применение встроенного класса `java.util.HashMap` для решения прикладной задачи — автоматизации учета товаров на складе.

Функционал:
Модель данных: Класс `Product` инкапсулирует данные о названии, цене и количестве товара.
Идентификация: В качестве ключа используется уникальный штрихкод товара (`String`).
Отображение: Переопределен метод `toString()` для формирования понятных отчетов о товарах.

Операции:
1. Добавление новых поступлений.
2. Поиск товара по штрихкоду.
3. Удаление списанных товаров.
4. Вывод полной инвентаризации склада.


Основные концепции

В ходе работы были применены и изучены следующие механизмы языка Java:

Хэширование: Преобразование объекта в числовой индекс для быстрого доступа к памяти.
Type Erasure (Стирание типов): Понимание того, как дженерики работают во время компиляции и исполнения.
Наследование от Object: Переопределение методов `equals()`, `hashCode()` и `toString()`.
Сложность алгоритмов**: Сравнение скорости работы списков (O(n)) и хэш-таблиц (O(1)).
Потокобезопасность: Изучены причины, по которым `HashMap` не предназначен для многопоточной среды, и альтернативы в виде `ConcurrentHashMap`.
